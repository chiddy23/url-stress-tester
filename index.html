URL stress tester

Use only on systems you own or have explicit permission to load-test. On Render, this page is served from the same app as the proxy — enable Use proxy, set Proxy base to this site’s origin (auto-filled on *.onrender.com), and paste your Proxy token (same value as STRESS_PROXY_TOKEN). For heavy prod tests, prefer k6 or wrk.

Local: node stress-proxy.mjs, open this file, Proxy base http://127.0.0.1:8765. Render: open your service URL (root /); Proxy base defaults to the same origin on *.onrender.com. Set STRESS_PROXY_TOKEN in the dashboard; paste it here for runs (not stored). Token is session-only after refresh. For company-only tests, set STRESS_PROXY_ALLOWED_HOST_SUFFIXES on the server (see README).

Max batch size is how many requests are in flight at once for each burst: the tool starts that many fetch calls together, waits until all of them finish, then starts the next burst (after the delay between bursts, if any). Raising the batch increases how hard you hit the URL in each wave; values that are too high can create more errors because of browser connection limits or an overloaded server. Allowed range: 1–256. Use the Max button next to the field to jump to 256 instantly.
